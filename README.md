# WB Position Bot

Отдельный проект для анализа позиций товаров Wildberries по поисковым запросам.

Старый Prom-бот не используется как зависимость и не меняется. Из него перенесена только общая архитектурная идея: env-конфиг, SQLite база, отдельные модули для клиента маркетплейса, анализа, отчетов и точки входа Railway.

## Что уже заложено

- хранение отслеживаемых карточек в SQLite;
- анализ выдачи Wildberries по запросу;
- топ-5 карточек по запросу;
- поиск позиции своей карточки по `nm_id`, ID продавца или названию магазина;
- сохранение истории проверок;
- недельный график изменения позиции с автоматическим переходом на новую неделю;
- общая статистика за все время сбора данных;
- PNG-инфографика для Telegram;
- текстовый отчет для Telegram/консоли;
- заготовка Telegram-команд;
- Railway `Procfile` и `railway.json`.

## Структура

```text
wb_position_bot/
  analyzer.py      # логика поиска позиции и топ-5
  analytics.py     # недельная/общая статистика и PNG-графики
  bot.py           # Telegram-слой
  config.py        # env-настройки
  db.py            # SQLite и миграции
  models.py        # dataclass-модели
  report.py        # форматирование отчета
  wildberries.py   # клиент публичной выдачи WB
main.py            # CLI и запуск бота
tests/             # базовые тесты анализа без сети
```

## Быстрый старт локально

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python main.py init-db
```

Добавить карточку:

```bash
python main.py add --nm-id 123456789 --query "поисковый запрос" --supplier "Название моего магазина"
```

Если артикул WB пока неизвестен, можно добавить только запрос и магазин:

```bash
python main.py add --query "1С:Бухгалтерия 8 для Казахстана. Базовая версия" --supplier "кодерлайн"
```

Запустить проверку одной карточки:

```bash
python main.py analyze --nm-id 123456789
```

Проверить все активные карточки:

```bash
python main.py analyze-all
```

Построить график текущей недели:

```bash
python main.py week --nm-id 123456789
```

Построить график и статистику за все время:

```bash
python main.py stats --nm-id 123456789
```

## Telegram

После добавления токена в `.env`:

```bash
python main.py bot
```

Первичная привязка:

```text
/start change-me
```

Минимальные команды:

```text
/status
/add 123456789 | поисковый запрос | Название моего магазина
/add поисковый запрос | Название моего магазина
/list
/check 123456789
/checkall
/week 123456789
/stats 123456789
/stats
```

## Недельная статистика

Каждая проверка сохраняется в `position_checks`. Текущая неделя считается с понедельника по воскресенье в `TIMEZONE`.

Когда начинается новая неделя, старые данные не удаляются. Команда `/week nm_id` просто строит график только по текущей неделе, а `/stats nm_id` берет всю историю за все недели.

На графике позиция `#1` находится сверху: чем выше линия, тем лучше карточка стоит в выдаче. Если карточка не найдена, это отмечается отдельной красной меткой в нижней части графика.

## Railway

Для Railway понадобятся переменные:

- `TELEGRAM_BOT_TOKEN`
- `SETUP_KEY`
- `DATABASE_PATH=data/wb_position_bot.db`
- `TIMEZONE=Europe/Kyiv`
- `REPORT_TIMES=09:00`
- `REPORT_INTERVAL_DAYS=1` или `2`
- `WB_DEST=-1257786`
- `WB_MAX_SEARCH_PAGES=20`
- `WB_REQUEST_DELAY_SECONDS=4`
- `WB_REQUEST_RETRIES=6`
- `WB_429_COOLDOWN_SECONDS=15`
- `WB_PROXY_URL=` необязательно, если Railway IP блокируется

Для SQLite на Railway нужен volume, смонтированный в `data/`, чтобы база не пропадала после деплоя.

## Важно по Wildberries

Публичная поисковая выдача WB может менять параметры и защиту. Поэтому `wildberries.py` сделан отдельным модулем: если WB изменит endpoint или понадобится официальный seller-analytics API, заменим только этот слой, а база, Telegram и отчеты останутся теми же.

По умолчанию бот смотрит до 20 страниц выдачи и останавливается раньше, если нашел нужную карточку. Между запросами есть задержка, повторы и отдельная пауза при HTTP 429.

Если Railway получает HTTP 429 даже на одиночном `/check`, значит WB ограничивает серверный IP. Тогда поставь в Railway Variables `WB_REQUEST_DELAY_SECONDS=4`, `WB_REQUEST_RETRIES=6`, подожди 5-10 минут и попробуй снова. Если 429 остается, нужен `WB_PROXY_URL` с российским HTTP/HTTPS proxy.

`WB_DEST=-1257786` оставлен как российский региональный параметр для проверки позиций. Если понадобится строго другой город/регион России, нужно будет заменить `WB_DEST` после теста на реальной выдаче WB.
